Routing is at the heart of any web framework, and Zuno keeps it simple and powerful. You define routes directly on the `zuno::App` instance using intuitive methods like `.get()`, `.post()`, `.put()`, `.patch()`, `.options()` and `.del()`.

Each route is associated with a path and a handler function that receives the request and response objects.

---

## 🧭 Basic Route Example

<Code title="main.cpp" lang="cpp" code={`zuno::App app;

app.get("/", [](auto req, auto res) {
  res.send("Welcome to Zuno!");
});

app.listen(8080);`} />

This creates a basic HTTP server that responds with `"Welcome to Zuno!"` when a GET request is made to `/`.

---

## 📦 Supported HTTP Methods

Zuno currently supports the following HTTP methods:

- `app.get(path, handler)`
- `app.post(path, handler)`
- `app.put(path, handler)`
- `app.del(path, handler)`

<Code title="methods.cpp" lang="cpp" code={`app.get("/info", [](auto req, auto res) {
  res.send("GET /info");
});

app.post("/submit", [](auto req, auto res) {
  res.send("POST /submit");
});

app.put("/update", [](auto req, auto res) {
  res.send("PUT /update");
});

app.patch("/update", [](auto req, auto res) {
  res.send("PATCH /update");
});

app.del("/remove", [](auto req, auto res) {
  res.send("DELETE /remove");
});`} />

---

## 🧠 Dynamic Route Parameters

Zuno supports dynamic route segments using the `:param` syntax, similar to Express.js. You can extract these parameters using `req.param("name")`.

<Code title="dynamic.cpp" lang="cpp" code={`app.get("/users/:id", [](auto req, auto res) {
  auto user_id = req.param("id");
  res.send("User ID: " + user_id);
});`} />

This route will match `/users/42` and respond with `"User ID: 42"`.

---

## 🧪 Handling Query Parameters

You can access query parameters using `req.query("key")`.

<Code title="query.cpp" lang="cpp" code={`app.get("/search", [](auto req, auto res) {
  auto query = req.query("q"); // e.g. /search?q=zuno
  res.send("Search: " + query);
});`} />

---

## 🧰 Organizing Routes

While Zuno does not yet support nested routers, you can organize your routes in separate files or functions for clarity:

<Code title="routes.cpp" lang="cpp" code={`void register_user_routes(zuno::App& app) {
  app.get("/users", [](auto req, auto res) {
    res.send("User list");
  });

  app.get("/users/:id", [](auto req, auto res) {
    res.send("User ID: " + req.param("id"));
  });
}`}/>

<Code title="main.cpp" lang="cpp" code={`#include "routes.cpp"

int main() {
  zuno::App app;
  register_user_routes(app);
  app.listen(8080);
}`}/>

---

## ⚠️ Things to Know

- Route matching is case-sensitive.
- Routes are matched in the order they are registered.
- If no route matches, Zuno returns a 404 by default.
