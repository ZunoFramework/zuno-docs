🧩 Extensions

Zuno includes a set of powerful built-in middleware extensions that you can enable with a single function call. These extensions simplify common tasks like compression, CORS, rate limiting, and serving static files.

Each extension is designed to be lightweight, configurable, and production-ready.

🗜️ Brotli Compression

Section titled “🗜️ Brotli Compression”

Enable Brotli compression to reduce response size and improve performance. You can configure the compression ratio (1–11).

  • Parameter: minRatio — Compression level (default: 0.8)

🌐 CORS (Cross-Origin Resource Sharing)

Section titled “🌐 CORS (Cross-Origin Resource Sharing)”

Enable CORS to allow cross-origin requests from browsers. You can customize allowed origins, methods, and headers.

  • Parameters:
    • origin: Allowed origin(s) (default: "*")
    • methods: Allowed HTTP methods
    • headers: Allowed request headers
  • Effect: Adds appropriate Access-Control-* headers to responses

🚦 Rate Limiting

Section titled “🚦 Rate Limiting”

Protect your app from abuse by limiting the number of requests per client over a time window.

  • Parameters:
    • max: Maximum number of requests per interval (default: 100)
    • intervalMs: Time window in milliseconds (default: 60,000 ms)
  • Effect: Returns 429 Too Many Requests if the limit is exceeded

🗂️ Static File Serving

Section titled “🗂️ Static File Serving”

Serve static assets like HTML, CSS, JS, or images from a directory. You can configure the URL prefix, caching, and max-age.

  • Parameters:

    • directory: Path to the static file directory (required)
    • urlPrefix: URL prefix to match (default: "/")
    • cache: Whether to enable caching headers (default: false)
    • maxAge: Cache duration in seconds (default: 3600)

  • Effect: Serves files like /public/index.html at http://localhost:8080/index.html

🧠 Best Practices

Section titled “🧠 Best Practices”
  • Use Brotli compression in production to reduce bandwidth.
  • Enable CORS only for trusted origins in secure environments.
  • Tune rate limits based on your app’s expected traffic.
  • Serve static files from a CDN in high-traffic deployments.
